Commands

Expected run order

source /opt/ros/humble/setup.bash
source ~/franka_ros2_ws/install/setup.bash

ros2 launch franka_fr3_moveit_config moveit.launch.py \
  robot_ip:=172.16.1.11 \
  use_fake_hardware:=false \
  load_gripper:=false

python src/validate/robot/franka_state.py
python src/capture/robot/franka_teaching.py
python src/capture/robot/franka_replay_check.py --step --home
python src/calibration/handeye/capture.py --arm franka
PYTHONPATH= python src/calibration/handeye/calculate.py --arm franka

If ROS imports fail with a GLIBCXX_3.4.30 error inside the conda env, prefix ROS Python commands with LD_PRELOAD=/usr/lib/x86_64-linux-gnu/libstdc++.so.6. franka_teaching.py also requires franka_msgs from a built and sourced franka_ros2 workspace.

Risks

Do not skip these checks

Validate DEVICE2WRIST["franka"] before teleop, allow robot UDP through ufw, keep the Desk end-effector load accurate, and expect real-time kernel issues if FCI reports communication constraint violations. Mixed conda/ROS environments may also need the LD_PRELOAD workaround above.
