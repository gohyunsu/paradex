Operating model

End-to-end pipeline map

Paradex is easiest to reason about as a chain of artifacts. Every stage either updates calibration state, captures raw synchronized data, derives processed assets, or consumes those assets for pose/grasp inference.

00

Prepare the rig

Install the editable package, confirm config files, bring up capture PCs, and verify robot network access.

Entry points
pip install -e ., src/camera/server_daemon.py, src/validate/
Output
Reachable camera daemons, known ports, current hardware config.
01

Calibrate

Run camera and robot calibration in dependency order. Re-run downstream steps whenever upstream geometry changes.

Order
intrinsic -> extrinsic -> handeye; optional xArm URDF patch.
Output
intrinsics.json, extrinsics.json, C2R.npy.
02

Capture

Collect synchronized images or videos, plus optional arm, hand, teleop, timestamps, and state histories.

Entry points
src/capture/camera/*_remote.py, src/dataset_acquisition/*/capture.py.
Output
~/shared_data/capture/.../raw/ session folders with camera params and C2R snapshots.
03

Process

Convert raw sessions into analysis-ready artifacts: frame sync, overlays, contact maps, masks, rotations, and COLMAP reconstructions.

Entry points
src/process/miyungpa/, src/process/object_turntable/.
Output
Synced robot streams, merged overlay videos, masks, rotations, sparse reconstructions.
04

Estimate pose

Undistort images, match object templates, optimize multi-view 6D pose, and validate C2R with robot overlays.

Entry points
src/object6d/capture.py, src/object6d/validate_c2r.py.
Output
obj_T_multiview_matching.pkl, validation grids, projected mesh overlays.

Infer and execute

Map grasps from object frame into robot frame, solve IK, then visualize or drive the GUI controller.

Entry points
src/inference/bodex/, src/inference/grasp_w_gui/.
Output
Projected sanity checks, Viser scenes, GUI stages for pregrasp, grasp, and squeeze.

Runtime topology

What runs where

The main PC coordinates. Capture PCs own cameras. Robot/teleop scripts run where the hardware is reachable. Shared storage is the contract between stages.

Main PC

  • Launches remote camera capture and stream clients over SSH.
  • Runs calibration solvers, processing orchestrators, pose estimation, and inference.
  • Writes camera parameter snapshots beside every capture.

Capture PCs

  • Run src/camera/server_daemon.py before remote capture.
  • Own local CameraLoader -> Camera -> PySpinCamera devices.
  • Expose ping, status, and command sockets on ports 5480, 5481, and 5482.

Robot + teleop

  • XArm, Allegro, Inspire, XSens, and Oculus are controlled by capture or dataset scripts.
  • CaptureSession can record camera, arm, hand, teleop, timestamps, and state together.
  • URDFs should be patched from factory xArm calibration before relying on FK/IK.

Storage contract

  • Raw sessions generally land under ~/shared_data/capture/....
  • Local heavy processing may use ~/paradex_download/... and upload results back.
  • Current calibration is loaded via paradex.calibration.utils.

Stage 1

Calibration dependency chain

Calibration is the most important ordering constraint in the repo. If focal length, camera placement, robot kinematics, or the end-effector fixture changes, refresh the dependent outputs before capturing new data.

Run order

  1. Intrinsic: produce per-camera internal parameters.
  2. Extrinsic: capture Charuco observations and solve multi-camera poses with COLMAP.
  3. xArm kinematics: optionally read factory parameters and patch URDFs.
  4. Hand-eye: drive the arm through a fixed trajectory and solve camera-to-robot C2R.npy.

Commands

# Extrinsic capture and solve, main PC
python src/calibration/extrinsic/capture.py
python src/calibration/extrinsic/calculate.py

# Optional xArm URDF patch
python src/calibration/xarm_kinematic_calibration.py --robot_ip 192.168.1.xxx

# Hand-eye capture and solve, main PC
python src/calibration/handeye/capture.py --arm xarm
python src/calibration/handeye/calculate.py --arm xarm

Extrinsic calibration

capture.py auto-launches capture-PC clients, displays detected and accumulated Charuco corners, and stores full-resolution images plus 2D markers. calculate.py builds a COLMAP database, solves camera poses, removes high reprojection-error corners, rescales to metric board geometry, then writes current camera params.

Extrinsic README

Hand-eye calibration

capture.py records image sets and robot state at every trajectory waypoint. calculate.py triangulates the board, computes FK, builds relative motions, solves Tsai-Lenz AX=XB, and renders debug overlays.

Stage 2

Capture modes

Camera-only capture can run locally or distributed. Full dataset capture uses CaptureSession to coordinate camera, robot, hand, teleop, sync, and metadata under one session directory.

Camera images

Keyboard-triggered still capture. Use distributed mode for the full rig.

# capture PCs
python src/camera/server_daemon.py

# main PC
python src/capture/camera/image_remote.py --save_path object6d/session1

Keys: c capture, q quit.

Camera video

Synchronized recordings. Enable hardware trigger when frame-level sync matters.

python src/capture/camera/video_remote.py \
  --save_path capture/object_turntable/mug \
  --sync_mode --fps 30

Keys: c start, s stop, q quit.

Teleop demos

XSens/Oculus driven robot demos, optionally recording arm and hand streams.

python src/dataset_acquisition/miyungpa/capture.py \
  --name demo1 --arm xarm --hand allegro --device xsens

Output includes video, arm, hand, teleop, timestamps, and state.

Turntable scans

Camera-only object video for reconstruction, mask extraction, and 6D assets.

python src/dataset_acquisition/object_turntable/capture.py --name mug

Rotate one full revolution between start and stop.

Stage 3

Post-processing pipelines

Processing turns raw capture folders into data products other stages can trust. The scripts are intentionally concrete and often hard-code dataset roots or object loops, so review the bottom of each script before running.

Miyungpa manipulation demos

Sync robot states to camera frames, render robot mesh overlays, create merged videos, compute contact artifacts, and serve galleries.

  1. download_dir copies raw data locally.
  2. match_sync aligns arm/hand streams to video frames.
  3. overlay projects robot meshes into each camera.
  4. upload_output sends processed artifacts back to shared storage.
# distributed
python src/process/miyungpa/process_client.py
python src/process/miyungpa/process_main.py

# viewers
python src/process/miyungpa/web.py
python src/process/miyungpa/visualizer.py

Object turntable reconstruction

Extract multi-camera frames, triangulate the Charuco turntable, solve per-frame rotations, segment object masks, and triangulate a COLMAP model.

  1. extract_video.py writes images/<serial>/frame_*.jpg.
  2. extract_charuco.py writes charuco_3d/.
  3. get_rotation.py writes rotation/<frame>.npy.
  4. extract_mask_sam3.py writes masks and masked images.
  5. generate_colmap.py writes colmap/database.db and sparse output.
python src/process/object_turntable/extract_video.py
python src/process/object_turntable/extract_charuco.py
python src/process/object_turntable/get_rotation.py
python src/process/object_turntable/extract_mask_sam3.py
python src/process/object_turntable/generate_colmap.py

Operational safety

Validation, recovery, and common checks

Use validation scripts as stage gates. They are faster than discovering bad calibration, dead capture daemons, frame drops, or robot alignment issues after a full recording session.

Before capture

  • Confirm every capture PC daemon responds on ports 5480-5482.
  • Run camera status monitor if the rig is unstable: src/camera/monitor_daemon.py.
  • Validate camera sync and hang recovery in src/validate/camera_system/.
  • Regenerate Charuco boards from src/util/marker/generate_charuco.py when board specs change.

When a run hangs

  • Use src/camera/reset_cameras.py from the main PC to relaunch capture daemons.
  • Check idle timeout behavior if the controller died mid-session.
  • Inspect timestamp and frame-id arrays before processing.
  • Use turntable check/ helpers to remove dropped or invalid frames before COLMAP.

Before robot execution

  • Validate C2R.npy with robot overlay grids.
  • Patch xArm URDFs with factory calibration for reliable FK/IK.
  • Preview grasp plans in Viser before opening the GUI controller.
  • Keep object marker-offset tables current under src/util/register_object/.
